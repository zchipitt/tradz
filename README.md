# Tradz - Multi-Source Trading Intelligence System
# Tradz - 多源交易情报系统

Daily automated trading signal generation powered by multi-source data aggregation and Claude AI.

基于多源数据聚合和 Claude AI 的每日自动交易信号生成系统。

---

## Features / 功能特性

### Core Data Sources / 核心数据源
- 📈 **US Equities / 美股**: Monitors major stocks using yfinance / 使用 yfinance 监控主要股票
- 💰 **Cryptocurrency / 加密货币**: Tracks top crypto pairs using ccxt / 使用 ccxt 追踪主流交易对
- 🏛️ **Congress Trading / 国会交易**: Tracks House & Senate member trades (45-day disclosure delay) / 追踪众议院和参议院成员交易（45天披露延迟）
- 🏦 **Hedge Fund 13F / 对冲基金13F**: Monitors institutional holdings from SEC EDGAR / 监控 SEC EDGAR 机构持仓
- 🎰 **Polymarket**: Prediction market odds for market-relevant events / 市场相关事件的预测市场赔率
- 📰 **News Aggregation / 新闻聚合**: Yahoo Finance + NewsAPI integration / Yahoo Finance + NewsAPI 集成
- 📋 **SEC Filings / SEC 文件**: 10-K, 10-Q, 8-K filing tracking / 10-K、10-Q、8-K 文件追踪

### Intelligence & Reporting / 智能分析与报告
- 🤖 **Claude AI Reports / Claude AI 报告**: High-quality reports generated by Claude Code CLI with MCP Skills / 通过 Claude Code CLI 和 MCP Skills 生成高质量报告
- 🔍 **Real-time Search / 实时搜索**: Claude uses Tavily to search latest news during report generation / Claude 使用 Tavily 在报告生成时搜索最新新闻
- 📊 **Cross-Source Analysis / 跨源分析**: Identifies patterns across multiple data sources / 识别多数据源间的模式
- 🎯 **Signal Generation / 信号生成**: Price movement, volatility, and volume analysis / 价格变动、波动率和成交量分析
- 📧 **Email Reports / 邮件报告**: Automated daily reports via SMTP / 通过 SMTP 自动发送每日报告

### Broker Integration (Optional) / 券商集成（可选）
- 💼 **IBKR Integration / 盈透证券集成**: Connect to Interactive Brokers for portfolio tracking / 连接盈透证券进行投资组合追踪

### Operations / 运维功能
- 🔒 **Dry-Run Mode / 测试模式**: Test without sending real emails / 测试运行不发送真实邮件
- ⏰ **Automated Scheduling / 自动调度**: Works with cron (Linux) and launchd (macOS) / 支持 cron (Linux) 和 launchd (macOS)
- 🔄 **Fallback Support / 降级支持**: Template-based reports when Claude is unavailable / Claude 不可用时使用模板报告

## Quick Start / 快速开始

### 1. Installation / 安装

```bash
# Clone or navigate to project directory / 克隆或进入项目目录
cd tradz

# Create virtual environment / 创建虚拟环境
python3 -m venv .venv

# Activate virtual environment / 激活虚拟环境
source .venv/bin/activate  # macOS/Linux
# or / 或
.venv\Scripts\activate     # Windows

# Install dependencies / 安装依赖
pip install -r requirements.txt
```

### 2. Configuration / 配置

#### A. Copy and configure environment file / 复制并配置环境文件:

```bash
cp .env.example .env
```

Edit `.env` with your SMTP settings / 编辑 `.env` 配置 SMTP:

```bash
# For Gmail (recommended) / Gmail 推荐配置:
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-specific-password
SMTP_FROM=your-email@gmail.com
SMTP_TO=your-email@gmail.com
DRY_RUN=1  # Set to 0 to actually send emails / 设为 0 发送真实邮件
```

**Important / 重要**: For Gmail, you must / 使用 Gmail 需要:
1. Enable 2-Factor Authentication / 启用两步验证
2. Generate an App Password / 生成应用专用密码: https://support.google.com/accounts/answer/185833
3. Use the App Password in `SMTP_PASS` / 在 `SMTP_PASS` 中使用应用专用密码

#### B. Customize watchlist (optional) / 自定义监控列表（可选）:

Edit `config.yaml` to add/remove tickers and crypto pairs / 编辑 `config.yaml` 添加或删除股票和加密货币:

```yaml
equities:
  tickers:
    - AAPL
    - GOOGL
    # Add more... / 添加更多...

crypto:
  pairs:
    - BTC/USDT
    - ETH/USDT
    # Add more... / 添加更多...
```

#### C. Configure data sources (optional) / 配置数据源（可选）:

Enable/disable data sources in `config.yaml` / 在 `config.yaml` 中启用或禁用数据源:

```yaml
# Congress member trading / 国会成员交易
congress:
  enabled: true
  lookback_days: 30
  min_amount: 15000

# Hedge fund 13F filings / 对冲基金 13F 文件
hedgefunds:
  enabled: true
  min_position_change_pct: 25.0

# Polymarket prediction markets / Polymarket 预测市场
polymarket:
  enabled: true
  categories: [Economics, Crypto, Business, Politics]

# News aggregation / 新闻聚合
news:
  enabled: true
  max_articles_per_ticker: 10

# SEC filings / SEC 文件
sec_filings:
  enabled: true
  form_types: [10-K, 10-Q, 8-K]

# Claude report generation / Claude 报告生成

  enabled: true
  timeout: 300
  fallback_to_template: true
```

### 3. Claude Code CLI Setup (Recommended) / Claude Code CLI 设置（推荐）

For AI-powered reports, install Claude Code CLI / 安装 Claude Code CLI 以启用 AI 报告:

```bash

npm install -g @anthropic-ai/claude-code

# Set your Anthropic API key in .env / 在 .env 中设置 Anthropic API 密钥

```

Claude will use MCP Skills (tavily-search, filesystem, etc.) to / Claude 将使用 MCP Skills 来:
- Search latest news for each signal / 搜索每个信号的最新新闻
- Compare with historical reports / 与历史报告对比
- Generate professional analyst-quality reports / 生成专业分析师级别的报告

### 4. Test Run (Dry-Run Mode) / 测试运行（测试模式）


## Automated Scheduling / 自动调度

### Linux (cron)

```bash
# Edit crontab / 编辑 crontab
crontab -e

# Add this line to run daily at 6:30 AM / 添加此行每天 6:30 运行
30 6 * * * /path/to/tradz/scripts/nightly.sh >> /path/to/tradz/logs/cron.log 2>&1

# Create logs directory / 创建日志目录
mkdir -p logs
```

**Example cron schedules / cron 调度示例:**
- `30 6 * * *` - Daily at 6:30 AM / 每天 6:30
- `0 7 * * 1-5` - Weekdays at 7:00 AM / 工作日 7:00
- `0 18 * * *` - Daily at 6:00 PM / 每天 18:00

### macOS (launchd)

Create `~/Library/LaunchAgents/com.tradz.nightly.plist` / 创建配置文件:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.tradz.nightly</string>

    <key>ProgramArguments</key>
    <array>
        <string>/path/to/tradz/scripts/nightly.sh</string>
    </array>

    <key>StartCalendarInterval</key>
    <dict>
        <key>Hour</key>
        <integer>6</integer>
        <key>Minute</key>
        <integer>30</integer>
    </dict>

    <key>StandardOutPath</key>
    <string>/path/to/tradz/logs/launchd.log</string>

    <key>StandardErrorPath</key>
    <string>/path/to/tradz/logs/launchd-error.log</string>

    <key>WorkingDirectory</key>
    <string>/path/to/tradz</string>
</dict>
</plist>
```

**Important / 重要**: Replace `/path/to/tradz` with your actual project path! / 将 `/path/to/tradz` 替换为实际项目路径！

Load the agent / 加载代理:
```bash
# Create logs directory / 创建日志目录
mkdir -p logs

# Load the agent / 加载代理
launchctl load ~/Library/LaunchAgents/com.tradz.nightly.plist

# Check if it's loaded / 检查是否已加载
launchctl list | grep tradz

# Unload (if needed) / 卸载（如需要）
launchctl unload ~/Library/LaunchAgents/com.tradz.nightly.plist
```

## Signal Methodology / 信号方法论

### Metrics Calculated / 计算指标

For each asset, we calculate / 对每个资产计算:

1. **Price Movement / 价格变动**
   - 1-day return (%) / 1日收益率
   - 7-day return (%) / 7日收益率

2. **Volatility / 波动率**
   - 7-day volatility (annualized) / 7日波动率（年化）
   - 30-day volatility (baseline) / 30日波动率（基准）
   - Volatility change (%) / 波动率变化

3. **Volume / 成交量**
   - Current volume vs 30-day average (ratio) / 当前成交量与30日均量比值

### Signal Scoring / 信号评分

Signals are scored 0-100 based on / 信号评分 0-100，基于:
- Strong price movements (>5% daily, >10% weekly) / 强劲价格变动（日>5%，周>10%）
- Volatility spikes (>25% increase vs baseline) / 波动率飙升（较基准>25%）
- Volume anomalies (>1.5x average) / 成交量异常（>1.5倍均量）

Top 5 signals per category (equities/crypto) are highlighted in the report.

每个类别（股票/加密货币）的前5个信号会在报告中突出显示。

## Troubleshooting / 故障排除

### Problem / 问题: "No module named 'yfinance'"

**Solution / 解决方案**: Make sure virtual environment is activated and dependencies are installed / 确保虚拟环境已激活且依赖已安装:
```bash
source .venv/bin/activate
pip install -r requirements.txt
```

### Problem / 问题: "SMTP authentication failed"

**Solutions / 解决方案**:
- Gmail: Use App Password, not your regular password / Gmail：使用应用专用密码，而非常规密码
- Check SMTP_USER and SMTP_PASS in `.env` / 检查 `.env` 中的 SMTP_USER 和 SMTP_PASS
- Verify SMTP_HOST and SMTP_PORT are correct for your provider / 验证 SMTP_HOST 和 SMTP_PORT 配置正确
- Test with `DRY_RUN=1` first to verify other components work / 先用 `DRY_RUN=1` 测试其他组件

### Problem / 问题: "Failed to fetch data for ticker XYZ"

**Causes / 原因**:
- Network connectivity issues / 网络连接问题
- Invalid ticker symbol / 无效的股票代码
- yfinance API rate limiting / yfinance API 限流
- Market data not available for that symbol / 该代码无市场数据

**Solutions / 解决方案**:
- Check ticker symbol is correct / 检查股票代码是否正确
- Remove problematic tickers from `config.yaml` / 从 `config.yaml` 中移除问题代码
- Run again later (rate limits reset) / 稍后重试（限流会重置）
- Check yfinance status / 检查 yfinance 状态: https://github.com/ranaroussi/yfinance

### Problem / 问题: "No exchange found" for crypto

**Causes / 原因**:
- ccxt cannot connect to configured exchange / ccxt 无法连接配置的交易所
- Exchange API is down / 交易所 API 宕机
- Network issues / 网络问题

**Solutions / 解决方案**:
- Change exchange in `config.yaml` (try 'coinbase' or 'kraken') / 在 `config.yaml` 中更换交易所
- Check exchange status / 检查交易所状态: https://status.binance.com
- Run with different trading pairs / 尝试不同的交易对

### Problem / 问题: Cron/launchd job not running

**Linux (cron)**:
```bash
# Check cron logs / 检查 cron 日志
grep CRON /var/log/syslog

# Test script manually / 手动测试脚本
cd /path/to/tradz && ./scripts/nightly.sh
```

**macOS (launchd)**:
```bash

launchctl list | grep tradz

# Check logs / 检查日志
cat logs/launchd.log
cat logs/launchd-error.log

# Test manually / 手动测试
./scripts/nightly.sh
```

### Problem / 问题: Reports directory is empty

**Check / 检查**:
1. Script runs without errors / 脚本运行无错误
2. Reports directory exists / 报告目录存在: `ls reports/`
3. Permissions / 权限: `ls -la reports/`
4. Check script output for error messages / 检查脚本输出的错误信息

## Data Sources & Limitations / 数据源与限制

### US Equities / 美股 (yfinance)
- **Source / 数据源**: Yahoo Finance
- **Delay / 延迟**: 15-20 minutes for free tier / 免费版延迟 15-20 分钟
- **Limitations / 限制**: Rate limiting, occasional data gaps / 限流，偶有数据缺失
- **Best for / 适用场景**: End-of-day analysis (not real-time trading) / 日终分析（非实时交易）

### Cryptocurrency / 加密货币 (ccxt)
- **Source / 数据源**: Multiple exchanges (Binance default, fallback to Coinbase/Kraken) / 多交易所（默认 Binance，备选 Coinbase/Kraken）
- **Delay / 延迟**: Near real-time / 近实时
- **Limitations / 限制**: Exchange-dependent, API rate limits / 依赖交易所，API 限流
- **Best for / 适用场景**: Daily trend analysis / 每日趋势分析

### Congress Trading / 国会交易
- **Source / 数据源**: House/Senate financial disclosures / 众议院/参议院财务披露
- **Delay / 延迟**: 45-day disclosure requirement / 45天披露要求
- **Limitations / 限制**: Delayed reporting, incomplete data / 报告延迟，数据不完整
- **Best for / 适用场景**: Following institutional/insider patterns / 跟踪机构/内部人士模式

### Hedge Fund 13F / 对冲基金 13F
- **Source / 数据源**: SEC EDGAR
- **Delay / 延迟**: Quarterly filings (45 days after quarter end) / 季度报告（季末后45天）
- **Limitations / 限制**: Only shows long positions >$100M AUM / 仅显示 AUM >1亿美元的多头持仓
- **Best for / 适用场景**: Tracking major institutional moves / 跟踪大型机构动向

### Polymarket
- **Source / 数据源**: Polymarket API
- **Delay / 延迟**: Near real-time / 近实时
- **Limitations / 限制**: Prediction market liquidity varies / 预测市场流动性不稳定
- **Best for / 适用场景**: Gauging market sentiment on events / 评估事件的市场情绪

### News Aggregation / 新闻聚合
- **Source / 数据源**: Yahoo Finance (default), NewsAPI (optional) / Yahoo Finance（默认），NewsAPI（可选）
- **Delay / 延迟**: Near real-time / 近实时
- **Limitations / 限制**: NewsAPI free tier: 100 requests/day / NewsAPI 免费版：每天100次请求
- **Best for / 适用场景**: Context for price movements / 价格变动的背景信息

### SEC Filings / SEC 文件
- **Source / 数据源**: SEC EDGAR
- **Delay / 延迟**: Filed within regulatory deadlines / 按监管期限提交
- **Limitations / 限制**: Requires SEC_USER_AGENT in .env / 需要在 .env 中配置 SEC_USER_AGENT
- **Best for / 适用场景**: Fundamental analysis, material events / 基本面分析，重大事件

⚠️ **Important / 重要**: This system uses free data sources and should NOT be used as the sole basis for trading decisions. Always verify signals with additional research.

本系统使用免费数据源，不应作为交易决策的唯一依据。请务必通过额外研究验证信号。

## Security Best Practices / 安全最佳实践

1. **Never commit `.env`** / **切勿提交 `.env`**: It contains your passwords / 包含您的密码
2. **Use App Passwords** / **使用应用专用密码**: For Gmail and similar services / 用于 Gmail 等服务
