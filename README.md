# Tradz - Multi-Source Trading Intelligence System

Daily automated trading signal generation powered by multi-source data aggregation and Claude AI.

## Features

### Core Data Sources
- 📈 **US Equities**: Monitors major stocks using yfinance
- 💰 **Cryptocurrency**: Tracks top crypto pairs using ccxt
- 🏛️ **Congress Trading**: Tracks House & Senate member trades (45-day disclosure delay)
- 🏦 **Hedge Fund 13F**: Monitors institutional holdings from SEC EDGAR
- 🎰 **Polymarket**: Prediction market odds for market-relevant events
- 📰 **News Aggregation**: Yahoo Finance + NewsAPI integration
- 📋 **SEC Filings**: 10-K, 10-Q, 8-K filing tracking

### Intelligence & Reporting
- 🤖 **Claude AI Reports**: High-quality reports generated by Claude Code CLI with MCP Skills
- 🔍 **Real-time Search**: Claude uses Tavily to search latest news during report generation
- 📊 **Cross-Source Analysis**: Identifies patterns across multiple data sources
- 🎯 **Signal Generation**: Price movement, volatility, and volume analysis
- 📧 **Email Reports**: Automated daily reports via SMTP

### Broker Integration (Optional)
- 💼 **IBKR Integration**: Connect to Interactive Brokers for portfolio tracking

### Operations
- 🔒 **Dry-Run Mode**: Test without sending real emails
- ⏰ **Automated Scheduling**: Works with cron (Linux) and launchd (macOS)
- 🔄 **Fallback Support**: Template-based reports when Claude is unavailable

## Quick Start

### 1. Installation

```bash
# Clone or navigate to project directory
cd tradz

# Create virtual environment
python3 -m venv .venv

# Activate virtual environment
source .venv/bin/activate  # macOS/Linux
# or
.venv\Scripts\activate     # Windows

# Install dependencies
pip install -r requirements.txt
```

### 2. Configuration

#### A. Copy and configure environment file:

```bash
cp .env.example .env
```

Edit `.env` with your SMTP settings:

```bash
# For Gmail (recommended):
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-specific-password
SMTP_FROM=your-email@gmail.com
SMTP_TO=your-email@gmail.com
DRY_RUN=1  # Set to 0 to actually send emails
```

**Important**: For Gmail, you must:
1. Enable 2-Factor Authentication
2. Generate an App Password: https://support.google.com/accounts/answer/185833
3. Use the App Password in `SMTP_PASS`

#### B. Customize watchlist (optional):

Edit `config.yaml` to add/remove tickers and crypto pairs:

```yaml
equities:
  tickers:
    - AAPL
    - GOOGL
    # Add more...

crypto:
  pairs:
    - BTC/USDT
    - ETH/USDT
    # Add more...
```

#### C. Configure data sources (optional):

Enable/disable data sources in `config.yaml`:

```yaml
# Congress member trading
congress:
  enabled: true
  lookback_days: 30
  min_amount: 15000

# Hedge fund 13F filings
hedgefunds:
  enabled: true
  min_position_change_pct: 25.0

# Polymarket prediction markets
polymarket:
  enabled: true
  categories: [Economics, Crypto, Business, Politics]

# News aggregation
news:
  enabled: true
  max_articles_per_ticker: 10

# SEC filings
sec_filings:
  enabled: true
  form_types: [10-K, 10-Q, 8-K]

# Claude report generation
claude:
  enabled: true
  timeout: 300
  fallback_to_template: true
```

### 3. Claude Code CLI Setup (Recommended)

For AI-powered reports, install Claude Code CLI:

```bash
# Install Claude Code CLI
npm install -g @anthropic-ai/claude-code

# Set your Anthropic API key in .env
ANTHROPIC_API_KEY=sk-ant-api03-xxxxx

# Verify installation
claude --version
```

Claude will use MCP Skills (tavily-search, filesystem, etc.) to:
- Search latest news for each signal
- Compare with historical reports
- Generate professional analyst-quality reports

### 4. Test Run (Dry-Run Mode)

```bash
# Run the nightly script
./scripts/nightly.sh

# Or run with specific options:
python -m src.tradz.run_nightly --skip-email        # Skip email
python -m src.tradz.run_nightly --template-only     # Use template, skip Claude
python -m src.tradz.run_nightly --use-claude        # Force use Claude
```

This will:
1. Fetch data from all sources (equities, crypto, congress, etc.)
2. Aggregate and save raw data to `data/`
3. Generate signals
4. Create report using Claude (or template fallback)
5. Save report files to `reports/`
6. Print email preview (not actually sent in dry-run mode)

Check outputs:
```bash
ls reports/
cat reports/$(date +%Y-%m-%d).md
```

### 4. Send Real Emails

Once you've verified everything works:

```bash
# Edit .env and set:
DRY_RUN=0

# Run again
./scripts/nightly.sh
```

## Automated Scheduling

### Linux (cron)

```bash
# Edit crontab
crontab -e

# Add this line to run daily at 6:30 AM
30 6 * * * /path/to/tradz/scripts/nightly.sh >> /path/to/tradz/logs/cron.log 2>&1

# Create logs directory
mkdir -p logs
```

**Example cron schedules:**
- `30 6 * * *` - Daily at 6:30 AM
- `0 7 * * 1-5` - Weekdays at 7:00 AM
- `0 18 * * *` - Daily at 6:00 PM

### macOS (launchd)

Create `~/Library/LaunchAgents/com.tradz.nightly.plist`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.tradz.nightly</string>

    <key>ProgramArguments</key>
    <array>
        <string>/path/to/tradz/scripts/nightly.sh</string>
    </array>

    <key>StartCalendarInterval</key>
    <dict>
        <key>Hour</key>
        <integer>6</integer>
        <key>Minute</key>
        <integer>30</integer>
    </dict>

    <key>StandardOutPath</key>
    <string>/path/to/tradz/logs/launchd.log</string>

    <key>StandardErrorPath</key>
    <string>/path/to/tradz/logs/launchd-error.log</string>

    <key>WorkingDirectory</key>
    <string>/path/to/tradz</string>
</dict>
</plist>
```

**Important**: Replace `/path/to/tradz` with your actual project path!

Load the agent:
```bash
# Create logs directory
mkdir -p logs

# Load the agent
launchctl load ~/Library/LaunchAgents/com.tradz.nightly.plist

# Check if it's loaded
launchctl list | grep tradz

# Unload (if needed)
launchctl unload ~/Library/LaunchAgents/com.tradz.nightly.plist
```

## Signal Methodology

### Metrics Calculated

For each asset, we calculate:

1. **Price Movement**
   - 1-day return (%)
   - 7-day return (%)

2. **Volatility**
   - 7-day volatility (annualized)
   - 30-day volatility (baseline)
   - Volatility change (%)

3. **Volume**
   - Current volume vs 30-day average (ratio)

### Signal Scoring

Signals are scored 0-100 based on:
- Strong price movements (>5% daily, >10% weekly)
- Volatility spikes (>25% increase vs baseline)
- Volume anomalies (>1.5x average)

Top 5 signals per category (equities/crypto) are highlighted in the report.

## Report Sections

Each daily report includes:

1. **Executive Summary** - 3-6 key takeaways
2. **Top Equity Signals** - Up to 5 highest-scoring stocks
3. **Top Crypto Signals** - Up to 5 highest-scoring crypto assets
4. **Watchlist Heatmap** - Table of all monitored assets
5. **Caveats & Data Quality** - Known limitations and risks
6. **What to Verify Tomorrow** - Follow-up items

## Troubleshooting

### Problem: "No module named 'yfinance'"

**Solution**: Make sure virtual environment is activated and dependencies are installed:
```bash
source .venv/bin/activate
pip install -r requirements.txt
```

### Problem: "SMTP authentication failed"

**Solutions**:
- Gmail: Use App Password, not your regular password
- Check SMTP_USER and SMTP_PASS in `.env`
- Verify SMTP_HOST and SMTP_PORT are correct for your provider
- Test with `DRY_RUN=1` first to verify other components work

### Problem: "Failed to fetch data for ticker XYZ"

**Causes**:
- Network connectivity issues
- Invalid ticker symbol
- yfinance API rate limiting
- Market data not available for that symbol

**Solutions**:
- Check ticker symbol is correct
- Remove problematic tickers from `config.yaml`
- Run again later (rate limits reset)
- Check yfinance status: https://github.com/ranaroussi/yfinance

### Problem: "No exchange found" for crypto

**Causes**:
- ccxt cannot connect to configured exchange
- Exchange API is down
- Network issues

**Solutions**:
- Change exchange in `config.yaml` (try 'coinbase' or 'kraken')
- Check exchange status: https://status.binance.com
- Run with different trading pairs

### Problem: Cron/launchd job not running

**Linux (cron)**:
```bash
# Check cron logs
grep CRON /var/log/syslog

# Test script manually
cd /path/to/tradz && ./scripts/nightly.sh
```

**macOS (launchd)**:
```bash
# Check if loaded
launchctl list | grep tradz

# Check logs
cat logs/launchd.log
cat logs/launchd-error.log

# Test manually
./scripts/nightly.sh
```

### Problem: Reports directory is empty

**Check**:
1. Script runs without errors
2. Reports directory exists: `ls reports/`
3. Permissions: `ls -la reports/`
4. Check script output for error messages

## Data Sources & Limitations

### US Equities (yfinance)
- **Source**: Yahoo Finance
- **Delay**: 15-20 minutes for free tier
- **Limitations**: Rate limiting, occasional data gaps
- **Best for**: End-of-day analysis (not real-time trading)

### Cryptocurrency (ccxt)
- **Source**: Multiple exchanges (Binance default)
- **Delay**: Near real-time
- **Limitations**: Exchange-dependent, API rate limits
- **Best for**: Daily trend analysis

⚠️ **Important**: This system uses free data sources and should NOT be used as the sole basis for trading decisions. Always verify signals with additional research.

## Security Best Practices

1. **Never commit `.env`**: It contains your passwords
2. **Use App Passwords**: For Gmail and similar services
3. **Rotate credentials**: Change passwords periodically
4. **Limit permissions**: Use email accounts dedicated to automation
5. **Monitor access**: Check email account for suspicious activity

## Customization

### Adding More Tickers

Edit `config.yaml`:
```yaml
equities:
  tickers:
    - AAPL
    - YOUR_NEW_TICKER
```

### Changing Signal Thresholds

Edit `config.yaml`:
```yaml
thresholds:
  day_return_high: 7.0  # Increase for fewer, stronger signals
  volume_high: 3.0      # Increase for more extreme volume alerts
```

### Changing Report Time

**Linux cron**: Edit crontab
```bash
crontab -e
# Change: 30 6 * * * to desired time
```

**macOS launchd**: Edit plist file
```xml
<key>Hour</key>
<integer>18</integer>  <!-- Change to desired hour -->
```
