## Codebase Patterns
- Use `SourceType` enum for observation sources (EQUITIES, CRYPTO, NEWS, SEC, CONGRESS, HEDGEFUND, POLYMARKET, BROKER, X_SENTIMENT)
- Use `EventType` enum for event classification (MARKET_ANOMALY, CATALYST_NEWS, CATALYST_FILING, FLOW_CONGRESS, FLOW_13F, PREDICTION_SHIFT, MIXED, UNCERTAIN)
- Use `EventStatus` enum for event lifecycle (NEW, ONGOING, STALE, RESOLVED, DISMISSED)
- Database singleton accessed via `get_database()` function
- 4D scoring model: anomaly_score, catalyst_score, flow_score, confidence_score (all 0-100)
- Attention score formula: 0.3*anomaly + 0.3*catalyst + 0.25*flow + 0.15*confidence + coverage_bonus
- Coverage bonus: +5 per unique source, max +20
- Events link to observations via `event_observations` table
- Use `datetime.now(timezone.utc)` instead of deprecated `datetime.utcnow()`
- LLM providers use abstract base class pattern with `generate()` method
- Title generation returns `tuple[str, str]` for (title, source) where source is 'llm' or 'template'
- Use `get_default_provider(config)` to auto-detect available LLM provider
- OpenRouter API key: `OPENROUTER_API_KEY` env var
- Tests use `from src.tradz.events import ...` pattern for imports
- Use `FactType` enum for fact type classification (PRICE, VOLUME, HEADLINE, POLITICIAN, etc.)
- Fact extraction via `extract_facts(observation)` dispatches to source-specific extractors
- Fact IDs use format: `{obs_id[:8]}_{fact_type}` for uniqueness
- API pattern: routers in `api/routers/`, schemas in `api/schemas/`, services in `api/services/`
- API tests use `from api.main import app` and `TestClient(app)` for HTTP testing (not router-specific)
- Register new routers in both `api/routers/__init__.py` and `api/main.py`
- Frontend API client in `frontend/src/api/client.ts` - add transformer functions for backend-to-UI mapping
- Frontend types in `frontend/src/api/types.ts` - add both backend schema types and legacy UI types
- Frontend hooks use TanStack Query with 5-min staleTime and refetchInterval for auto-refresh
- Use `EventCardSkeleton` component pattern for loading states with `animate-pulse` class
- Pass `isLoading`, `isError`, `error`, `onRetry` props to components for proper state handling
- Action labels use confidence thresholds: >=70 = Act (green), >=40 = Investigate (yellow), <40 = Monitor (gray)
- Hover effects with brutalist style: `hover:shadow-[4px_4px_0px_0px_rgba(0,0,0,0.8)]` + translate
- Event actions use `POST /api/events/{event_id}/actions` with `action` field (pin, unpin, snooze, dismiss, resolve)
- Optimistic updates pattern: `onMutate` cancels queries + caches previous data, `onError` rolls back, `onSettled` invalidates
- Timeline API uses `TimelineSourceFilter` enum for source filtering: all, market (equities+crypto), news, sec, congress, 13f, polymarket
