## Codebase Patterns
- Use `SourceType` enum for observation sources (EQUITIES, CRYPTO, NEWS, SEC, CONGRESS, HEDGEFUND, POLYMARKET, BROKER, X_SENTIMENT)
- Use `EventType` enum for event classification (MARKET_ANOMALY, CATALYST_NEWS, CATALYST_FILING, FLOW_CONGRESS, FLOW_13F, PREDICTION_SHIFT, MIXED, UNCERTAIN)
- Use `EventStatus` enum for event lifecycle (NEW, ONGOING, STALE, RESOLVED, DISMISSED)
- Database singleton accessed via `get_database()` function
- 4D scoring model: anomaly_score, catalyst_score, flow_score, confidence_score (all 0-100)
- Attention score formula: 0.3*anomaly + 0.3*catalyst + 0.25*flow + 0.15*confidence + coverage_bonus
- Coverage bonus: +5 per unique source, max +20
- Events link to observations via `event_observations` table
- Use `datetime.now(timezone.utc)` instead of deprecated `datetime.utcnow()`
