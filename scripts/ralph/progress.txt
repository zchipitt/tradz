## Codebase Patterns
- Database migrations: Use `_run_migrations()` method in Database class with try/except for ALTER TABLE
- Model fields: Add new fields to dataclasses with defaults to maintain backward compatibility
- Row conversion: Use `len(row) > N` pattern to handle both old and new schema rows
- Enum extensions: New enum values can be added without breaking existing data
- Foreign keys: Make nullable (no foreign key constraint) if record may not exist yet
- Datetime: Use `datetime.now(timezone.utc)` not deprecated `datetime.utcnow()`
- Unused parameters: Prefix with underscore (`_param`) to satisfy linters while keeping interface
