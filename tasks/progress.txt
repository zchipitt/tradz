## Codebase Patterns
- Use brutalist design aesthetic for event components: black/white + yellow accent, border-2 border-black, uppercase text for headers
- Event components follow consistent file structure: props interface → helper functions → sub-components → main export
- FactEntry types in api/types.ts match backend FactEntry Pydantic schema (fact_id, fact_type, label, value, unit, source, timestamp)
- Use cn() from lib/utils for conditional class names (imported from '../../lib/utils' in components)
- Event components export from frontend/src/components/events/index.ts
- TimelineObservation already includes fact_entries, so FactSpotlight can get all facts from observations prop
- Use lucide-react for icons throughout the frontend
